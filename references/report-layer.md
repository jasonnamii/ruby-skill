# Report Layer — 3-Layer 리포트 구조

러비스킬 산출물을 **독자 관점 역순**으로 구성하는 프레임.
파이프라인 실행 순서(Phase 0→5)와 리포트 출력 순서를 분리한다.

---

## 설계 원리

| 원리 | 내용 |
|------|------|
| **독자 역순** | 분석자 순서(분해→검증→판정→설계)가 아니라, 의사결정자 순서(판정→설계→근거)로 출력 |
| **판정 문장 필수** | 6축 점수표만으로는 "그래서 뭘 어쩌라고?"에 답하지 못함. 병목 축과 레벨 갭을 한 문장으로 종합 |
| **분석 과정 ≠ 산출물** | Phase 1~5의 중간 결과물은 근거 레이어에 격납. 독자는 필요할 때만 참조 |

---

## 3-Layer 구조

### Layer 1: 판정 (Verdict)

**위치**: 리포트 최상단
**분량**: 3~5문장

| 항목 | 내용 | 예시 |
|------|------|------|
| 판정 문장 | 병목 축 1~2개를 콕 집어, 현재 레벨과 목표 레벨 갭을 한 문장으로 | "현재 L2(간결). **사용적합성 ✕**가 L3 도달을 막고 있다 — 독자의 실제 워크플로우와 문서 구조가 동형이 아님." |
| 현재 Level | L0~L4 중 하나 (beauty-criteria 기준) | L2 |
| 목표 Level | 최소 L2, 가능하면 L3~L4 | L3 |
| 병목 축 | 6축 중 가장 낮은 축 (복수 가능) | 사용적합성 ✕, 방향성 △ |
**판정 문장 생성 로직**:
1. Phase 3(6축 판정)에서 ✕ 또는 최저 △ 축을 식별
2. beauty-criteria의 "병목 축별 상위 설계 방향" 테이블에서 해당 방향 추출
3. 현재 Level과 목표 Level 갭을 명시
4. 위 3요소를 한 문장으로 종합: "[현재 Level]. [병목 축]이 [목표 Level] 도달을 막고 있다 — [구체적 증상]."

---

### Layer 2: 설계안 (Design Proposal)

**위치**: 판정 바로 다음
**분량**: 리포트 본문의 중심

| 항목 | 내용 |
|------|------|
| 병목 해결안 | 병목 축별 구체적 변경안. 가장 낮은 축부터 |
| Before/After | 원본 텍스트(또는 코드)의 **실제 발췌**와 제안의 **실제 발췌**를 나란히 비교 |
| 논리 근거 | 러셀 분해·비트겐슈타인 검증 결과를 **직접 나열하지 않고**, "왜 이 변경이 작동하는가"의 논거로만 인용 |
| 논리 4요소 통합 | 연역(기존 원리에서 도출)·귀납(패턴 압축)·가추(새 구조 가설)·확률(맥락 적합성)이 설계 근거에 자연스럽게 녹아듦 |
| 디테일 보존 매트릭스 | **필수** — 원본 모든 원자 요소의 운명(유지/변환/제거+사유) |

**핵심**: 분해표·검증 체크리스트를 독립 섹션으로 나열하지 않는다. Before/After + 논거 서술 안에서 필요한 만큼만 인용.
---

### Layer 3: 분석 근거 (Evidence Appendix)

**위치**: 리포트 하단 (접이식/부록)
**분량**: 필요한 만큼

| 포함 항목 | 출처 Phase |
|-----------|-----------|
| 러셀 분해 매핑 테이블 | Phase 1 |
| 비트겐슈타인 8항 검증 결과 | Phase 2 |
| 6축 상세 점수표 | Phase 3 |
| 자기검증 결과 | Phase 5 |

**출력 형식**:
- 대화 응답: 접이식(콜아웃 또는 details/summary)으로 제공
- .md 파일 산출물: `## 부록: 분석 근거` 섹션으로 하단 배치
- 옵시디언: `> [!info]- 분석 근거` 접이식 콜아웃

**디테일 보존 원칙은 이 층에서 완전히 충족**: 매핑 테이블의 모든 원본 요소가 여기에 존재해야 한다. 누락 행 = FAIL.

---

## 모드별 Layer 적용

| 모드 | Layer 1 | Layer 2 | Layer 3 |
|------|---------|---------|---------|
| **진단** | 현재 Level + 병목 축 요약 (판정 문장) | — (설계안 없음) | 분해 매핑 + 6축 진단 |
| **평가** | 판정 문장 + 이중 검증 요약 | — (설계안 없음) | 분해 매핑 + 검증 결과 + 6축 상세 |
| **상위설계** | 판정 문장 (풀) | 설계안 (풀) | 전체 근거 (풀) |
---

## Gotchas

- **Layer 1 스킵 금지**: 6축 점수표를 먼저 보여주고 판정을 뒤에 놓으면 다시 데이터 시트가 됨. 반드시 판정 문장이 첫 줄
- **Layer 2에서 분석 과정 유출 주의**: "러셀 분해 결과에 따르면…"식의 도구 명칭 직접 노출보다, "구조 분해 시 X가 드러났다"처럼 논거로 녹여야 함
- **Layer 3 생략 금지**: 근거 없는 판정은 신뢰를 잃음. 접이식이라도 반드시 포함
- **디테일 보존과 Layer 분리의 양립**: 디테일 보존 매트릭스는 Layer 2에, 분해 매핑 전체는 Layer 3에. 두 곳에서 교차 참조 가능해야 함
