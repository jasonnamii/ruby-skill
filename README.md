# ruby-skill

**Logical decomposition and meaning verification: refactor code, documents, and content into beautiful higher-level design.**

## Goal

ruby-skill solves the problem of design debt and unclear thinking. Using Russell (structural decomposition — cut to atomic components) and Wittgenstein (meaning verification — does each part say what it should?), it transforms tangled implementations into clarified architecture, unclear docs into precision prose, and muddled content into sharp narrative.

## When & How to Use

Deploy when refactoring code, reworking documentation, or elevating clarity of existing work. Most powerful as post-draft refinement. Unlike TRIZ (contradictions), ruby-skill focuses on coherence and elegance. Works across code, markdown, and content.

## Use Cases

| Scenario | Prompt | What Happens |
|---|---|---|
| Messy codebase | `"Decompose this 500-line module into atomic functions"` | Russell decomposition→Wittgenstein verification→clear hierarchy, minimal coupling |
| Dense documentation | `"Refactor this 20-section guide into precise atomic sections"` | Atomic claims→each section proves one point→unified narrative |
| Meandering content | `"Elevate: decompose arguments, verify claims, resurface with arc"` | Extract claims→check evidence→reweave coherent narrative |

## Key Features

- Russell decomposition: cut to smallest meaningful atomic units
- Wittgenstein verification: each component performs exactly its stated function
- Dual-frame cycle: Decompose → Verify → Resurface as unified design
- Multi-domain: code, documents, content, data structures, process flows
